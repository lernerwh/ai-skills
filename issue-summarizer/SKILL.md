# 问题单总结生成器

将代码问题/缺陷批量转化为结构化的总结文档，用于构建代码检视知识库，避免同类问题重犯。

## 输入格式

### 格式一：CSV文件（推荐）

CSV文件必须包含以下列（列名不区分大小写）：

| 列名 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 问题单号 | 是 | 问题唯一标识（DTS+日期+序号） | `DTS2026032600001` |
| 问题描述 | 是 | 问题的详细描述 | `用户登录时密码明文记录到日志` |
| diff文件 | 是 | diff文件的绝对路径 | `D:\project\fixes\DTS2026032600001.diff` |
| 严重程度 | 否 | Critical/High/Medium/Low | `Critical` |
| 文件路径 | 否 | 问题所在的源文件路径 | `src/auth/login.ts` |
| 行号 | 否 | 问题所在行号 | `42` |

**问题单号格式**：`DTS` + `YYYYMMDD` + `4位序号`，如 `DTS2026032600001`

**CSV示例：**
```csv
问题单号,问题描述,diff文件,严重程度,文件路径,行号
DTS2026032600001,用户登录时密码明文记录到日志,D:\project\fixes\DTS2026032600001.diff,Critical,src/auth/login.ts,42
DTS2026032600002,订单查询存在SQL注入漏洞,D:\project\fixes\DTS2026032600002.diff,Critical,src/order/dao.ts,78
DTS2026032600003,文件上传后未关闭文件流导致资源泄漏,D:\project\fixes\DTS2026032600003.diff,High,src/upload/handler.ts,156
```

### 格式二：DTS单号 + 描述 + diff（路径或内容）

直接提供DTS单号、描述和diff信息，支持两种方式：

**方式A：diff文件路径**
```
DTS2026032600001
描述：用户登录时密码明文记录到日志
Diff文件：D:\fixes\DTS2026032600001.diff
```

**方式B：diff内容直接提供**
```
DTS2026032600001
描述：用户登录时密码明文记录到日志
Diff内容：
--- a/src/auth/login.ts
+++ b/src/auth/login.ts
@@ -40,7 +40,7 @@
-    Logger.info('User login: ' + password);
+    Logger.info('User login: [REDACTED]');
```

**对话中列出多个：**
```
请总结以下问题：

1. DTS2026032600001 - 日志输出明文密码
   Diff: D:\fixes\DTS2026032600001.diff

2. DTS2026032600002 - SQL注入风险
   Diff内容：
   --- a/dao.ts
   +++ b/dao.ts
   -const sql = `SELECT * FROM users WHERE id = ${userId}`;
   +const sql = 'SELECT * FROM users WHERE id = ?';
```

### 格式三：diff文件（文件名含DTS单号和描述）

直接提供一个或多个diff文件，文件名格式包含DTS单号和描述：

**文件名格式**：
- `DTS2026032600001_日志输出明文密码.diff`
- `DTS2026032600001-日志输出明文密码.patch`
- `DTS2026032600001 日志输出明文密码.diff`

**分隔符支持**：`_`（下划线）、`-`（短横线）、` `（空格）

**使用方式**：
```
用户: 帮我分析这个diff文件：D:\fixes\DTS2026032600001_日志输出明文密码.diff

用户: 请处理 D:\fixes\ 目录下所有的diff文件，文件名格式是 DTS单号_描述.diff
```

**批量处理目录**：
```
用户: 处理 D:\fixes\ 目录下的所有diff文件
文件名格式：DTS单号_问题描述.diff
```

## 工作流程

```
识别输入格式 → 解析问题信息 → 拆分为独立问题 → 并行启动subagent分析 → 生成独立md文档 → 创建索引文件
```

### 步骤 1: 识别输入格式并解析

**1.1 CSV文件格式**
- 读取CSV文件并解析为问题列表
- 提取：问题单号、问题描述、diff文件路径

**1.2 DTS单号+描述+diff格式**
- 从对话中解析问题信息
- 识别diff是文件路径还是直接内容
- 如果是路径，后续读取文件内容；如果是内容，直接使用

**1.3 diff文件（文件名含DTS信息）**
- 解析文件名，提取DTS单号和描述
- 文件名格式：`DTS2026032600001_问题描述.diff`
- 支持的分隔符：`_`、`-`、` `（空格）
- 正则匹配：`^(DTS\d{13})[_\-\s](.+)\.(diff|patch)$`

**1.4 验证**
- 验证DTS单号格式：`DTS` + 8位日期 + 4位序号
- 确认diff内容或文件存在
- 统计待处理问题数量

### 步骤 2: 制定处理计划

创建问题处理计划：
```
## 问题处理计划

| 序号 | 问题单号 | diff文件 | 状态 |
|------|---------|---------|------|
| 1 | DTS2026032600001 | DTS2026032600001.diff | pending |
| 2 | DTS2026032600002 | DTS2026032600002.diff | pending |
| ... | ... | ... | ... |

总计: X 个问题待处理
```

### 步骤 3: 并行分析问题

**对于每个问题，并行启动一个subagent执行分析：**

```typescript
task(
  category="unspecified-high",
  load_skills=[],
  run_in_background=true,
  description="分析问题 {issue_id}",
  prompt="""
## 任务
分析以下代码问题并生成结构化总结。

  
## 问题信息
- **问题单号**: {issue_id}
- **问题描述**: {description}
- **严重程度**: {severity}
- **相关文件**: {file_path}
- **行号**: {line_number}

## Diff内容
请读取并分析: {diff_file_path}

## 分析要求
1. 仔细阅读diff内容，理解问题本质和修复方案
2. 分析问题产生的原因（根本原因而非表面现象）
3. 提炼问题类型关键词（2-5个）
4. 生成可操作的检视要点

## 输出格式
严格按照以下格式输出（不要添加任何额外说明）：

---ISSUE_SUMMARY_START---
**问题单号**: {issue_id}
**问题类型关键词**: 关键词1, 关键词2, 关键词3
**严重程度**: {severity}
**根本原因**: 一句话描述问题产生的根本原因
**问题分析**: 2-3句话分析问题本质
**修复要点**: 
- 要点1
- 要点2
- 要点3
**检视要点**: 
- 检视时如何发现此类问题
- 问题代码特征
- 建议的检查方式
---ISSUE_SUMMARY_END---
"""
)
```

**重要**: 
- 所有问题的subagent应同时并行启动（不要串行）
- 每个subagent独立分析，不依赖其他问题的结果

### 步骤 4: 收集分析结果

1. 等待所有subagent完成（系统会通知）
2. 使用 `background_output(task_id)` 收集每个任务的结果
3. 提取 `---ISSUE_SUMMARY_START---` 和 `---ISSUE_SUMMARY_END---` 之间的内容

### 步骤 5: 生成独立md文档

**对于每个问题，生成独立的md文档，文件名格式：`{问题单号}问题总结.md`**

文档模板：
```markdown
# {问题单号} 问题总结

## 基本信息
| 属性 | 值 |
|------|-----|
| 问题单号 | {issue_id} |
| 严重程度 | {severity} |
| 问题类型 | {keywords} |

## 问题描述
{原始问题描述}

## 根本原因
{从分析结果提取}

## 问题分析
{从分析结果提取}

## 修复要点
{从分析结果提取的修复要点列表}

## 检视要点
{从分析结果提取的检视要点}

## 相关文件
- 源文件: {file_path}
- Diff文件: {diff_file}

---
*生成时间: {timestamp}*
```

### 步骤 6: 创建索引文件

生成 `问题总结索引.md` 汇总所有问题：

```markdown
# 问题总结索引

## 统计
- 问题总数: X 个
- Critical: X 个
- High: X 个
- Medium: X 个
- Low: X 个

## 问题列表

| 问题单号 | 严重程度 | 关键词 | 总结文件 |
|---------|---------|--------|---------|
| DTS2026032600001 | Critical | 日志安全, 敏感信息 | [DTS2026032600001问题总结.md](./DTS2026032600001问题总结.md) |
| DTS2026032600002 | Critical | SQL注入, 参数化查询 | [DTS2026032600002问题总结.md](./DTS2026032600002问题总结.md) |

## 关键词索引

### 按类型分类
- **安全漏洞** (X个)
  - DTS2026032600001: 日志安全
  - DTS2026032600002: SQL注入
- **性能问题** (X个)
  - DTS2026032600003: 资源泄漏
- **代码规范** (X个)
  - ...

## 常见问题模式

### 1. 日志安全问题
**特征**: 在日志中输出敏感信息
**检查方式**: 搜索日志语句中的密码、token等关键字
**相关问题**: DTS2026032600001, DTS2026032600005

### 2. SQL注入
**特征**: 直接拼接SQL字符串
**检查方式**: 检查SQL语句是否使用参数化查询
**相关问题**: DTS2026032600002, DTS2026032600007

---
*生成时间: {timestamp}*
```

## 问题类型分类参考

| 类型 | 说明 | 示例关键词 |
|------|------|-----------|
| 安全漏洞 | 可能导致数据泄露或被攻击 | `SQL注入`, `XSS`, `敏感信息泄露`, `越权访问` |
| 性能问题 | 影响系统响应速度或资源消耗 | `循环优化`, `内存泄漏`, `N+1查询`, `缓存缺失` |
| 代码规范 | 违反编码标准或最佳实践 | `命名规范`, `注释缺失`, `代码重复`, `函数过长` |
| 逻辑错误 | 代码逻辑不正确 | `边界条件`, `空值处理`, `类型错误`, `状态错误` |
| 资源管理 | 资源未正确释放或管理 | `资源泄漏`, `连接未关闭`, `流未关闭`, `锁未释放` |
| 并发问题 | 多线程/异步相关问题 | `竞态条件`, `死锁`, `线程安全`, `原子性` |
| 错误处理 | 异常处理不当 | `异常吞没`, `错误传播`, `重试缺失`, `超时处理` |

## 严重程度定义
| 级别 | 含义 | 示例 |
|------|------|------|
| **Critical** | 必须修复，可能导致严重后果 | 安全漏洞、数据丢失、系统崩溃 |
| **High** | 应该优先修复 | 资源泄漏、功能缺陷、数据不一致 |
| **Medium** | 建议修复 | 性能问题、代码质量、可维护性 |
| **Low** | 可选修复 | 代码风格、文档完善、优化建议 |

## 输出文件结构

```
{output_dir}/
├── DTS2026032600001问题总结.md
├── DTS2026032600002问题总结.md
├── DTS2026032600003问题总结.md
├── ...
└── 问题总结索引.md
```

## 注意事项

1. **并行处理**：多个问题必须并行启动subagent，提高处理效率
2. **diff文件**：diff文件路径必须是绝对路径
3. **格式一致性**：确保所有总结文档格式统一
4. **关键词提炼**：关键词应简洁（2-5个）、准确，便于后续检索
5. **实用性优先**：检视要点要具体可操作，避免空泛描述
6. **索引完整性**：索引文件必须包含所有问题的链接和关键词

## 使用示例

```
用户: 帮我处理 D:\reports\code_issues.csv，生成问题总结
输出到 D:\reports\summaries 目录
```

### 示例2: 对话中列出
```
用户: 请总结以下问题：
1. DTS2026032600001: 用户密码明文存储 - diff: D:\fixes\DTS2026032600001.diff
2. DTS2026032600002: 查询无分页导致内存溢出 - diff: D:\fixes\DTS2026032600002.diff
3. DTS2026032600003: 异常被吞没未记录 - diff: D:\fixes\DTS2026032600003.diff

输出到: D:\project\issue_summaries
```

```

帮我生成每个问题的总结文档，输出到 D:\reviews\summaries 文件夹
```

# hmos-code-reviewer

鸿蒙(HarmonyOS)/ArkTS 代码 AI 检视技能，基于预定义的 checklist 进行深度代码分析，生成结构化的 CSV 检视报告。

## 功能特性

- **全面覆盖**: 8 大检视维度，涵盖代码风格、安全性、性能、可维护性等
- **鸿蒙专属**: 针对 ArkUI/ArkTS 特性优化，包含状态管理、生命周期等检查项
- **并行处理**: 支持大规模代码库的并行检视，提升效率
- **标准输出**: 生成 CSV 格式的结构化报告，便于集成和分析

## 快速开始

### 检视单个文件

```
检视一下 src/main/ets/pages/Home.ets
```

### 检视目录

```
对 src/main/ets/components 目录下的代码进行检视
```

### 检视整个项目

```
帮我检视整个鸿蒙项目的代码质量
```

## 输出格式

检视结果以 CSV 格式输出，包含以下字段：

| 字段 | 说明 |
|------|------|
| 问题描述 | 具体的问题描述 |
| 文件路径 | 代码文件完整路径 |
| 行号范围 | 如 "42" 或 "100-120" |
| 严重程度 | Critical / High / Medium / Low |
| 检视依据 | 对应 checklist 中的检查项 |

## 严重程度定义

| 级别 | 含义 | 示例 |
|------|------|------|
| **Critical** | 必须修复，可能导致崩溃或安全漏洞 | 空指针解引用、敏感数据泄露 |
| **High** | 应该修复，影响功能或稳定性 | 资源泄漏、未处理的异常 |
| **Medium** | 建议修复，影响代码质量 | 代码重复、复杂度过高 |
| **Low** | 可选修复，优化建议 | 注释缺失、格式问题 |

## 检视维度

基于 `references/checklist.md` 进行全面检查：

1. **代码风格与规范** - 命名、格式、注释
2. **安全性问题** - 输入验证、数据安全、空值检查
3. **性能问题** - 渲染性能、内存管理、异步处理
4. **可维护性** - 代码组织、函数设计、依赖管理
5. **错误处理** - 错误捕获、恢复、日志记录
6. **ArkUI/ArkTS 特定** - 组件设计、状态管理、生命周期
7. **业务逻辑** - 逻辑正确性、用户体验
8. **测试相关** - 可测试性、测试覆盖

## 工具脚本

### 合并报告

使用 `scripts/merge_reports.py` 合并多个检视报告：

```bash
# 合并目录下所有 CSV 文件
python scripts/merge_reports.py ./reports/

# 合并指定文件并指定输出路径
python scripts/merge_reports.py report1.csv report2.csv -o final_report.csv
```

## 目录结构

```
hmos-code-reviewer/
├── SKILL.md              # 技能定义（触发条件、工作流程）
├── README.md             # 本文档
├── references/
│   └── checklist.md      # 检视清单（8大维度）
└── scripts/
    └── merge_reports.py  # 报告合并脚本
```

## 适用场景

- 代码提交前的自我检视
- 团队代码审查
- 项目代码质量评估
- 持续集成中的代码检查

## 注意事项

1. 检视前会完整读取目标文件内容
2. 大规模代码会自动拆分为多个并行任务
3. 只报告确定存在的问题，避免误报
4. 严重程度评估客观，参考 checklist 映射指南
